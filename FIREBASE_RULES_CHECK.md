# Firebase 규칙 확인 체크리스트

## 현재 상황
- 규칙은 올바르게 입력되어 있다고 함
- 하지만 테스트 결과: **모든 경로에서 401 Permission denied**
- 이것은 **규칙이 실제로 적용되지 않고 있음**을 의미

## 확인해야 할 사항

### 1. 규칙이 게시되었는지 확인
Firebase Console에서:
1. Realtime Database → 규칙 탭
2. 상단에 **"게시됨"** 또는 **"게시되지 않은 변경사항"** 표시 확인
3. **"게시되지 않은 변경사항"**이 보이면 → **"게시"** 버튼 클릭

### 2. 규칙 형식 재확인
규칙 편집기에 정확히 다음이 있어야 합니다:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

**주의사항:**
- 첫 줄: `{` (빈 줄 없이)
- `"rules"` 키가 반드시 있어야 함
- `.read`와 `.write`는 `true` (문자열이 아님)
- 마지막 줄: `}`

### 3. 규칙 게시 후 확인
규칙을 게시한 후:
1. 브라우저 새로고침 (F5)
2. 규칙 탭에서 "게시됨" 상태 확인
3. 아래 테스트 스크립트 실행:
   ```bash
   python scripts/test_firebase_rules.py
   ```
4. 모든 테스트가 **200 OK**를 반환해야 함

### 4. 규칙이 적용되지 않는 경우
- 규칙을 삭제하고 다시 입력
- 다른 브라우저에서 시도
- Firebase Console 로그아웃 후 다시 로그인
- 몇 분 후 다시 시도 (규칙 적용에 시간이 걸릴 수 있음)
